Collapse

Collapsible components are a list of headers that can be clicked to hide or reveal additional content.

  • Code header areas are collapsible sections as button so they are usable with both screen readers and keyboards.
  • Buttons should state if they are expanded with aria-expanded="true"
  • Each content area will have its aria-hidden attribute set to either true or false by the component, depending on its corresponding button's aria-expanded attribute. To ensure that your content is accessible in the event that JavaScript does not load or is disabled, you should set aria-hidden="true" on any of your content areas.

Specifications

The collapse panes will default to open with the option of collapsing each section.

On the right hand of the header there will be an arrow icon, which will change according to whether or not the section is open or closed. See below for an example.

Usability

When to use
  • Use collapsible components to organize case view information (i.e. case, party, event, financial, etc.).
  • Due to a large quantity of information in a single case view, a collapsible pane allows users to expand the information needed immediately and hide information not needed at the moment.
Guidance
  • Allow users to click anywhere in the header area to expand or collapse content.
  • Make sure interactive elements within the collapsible region are far enough from the headers that users don't accidentally trigger a collapse.
