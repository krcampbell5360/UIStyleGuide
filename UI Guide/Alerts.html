Alerts/Notifications

Alerts notify the user that an important item/issue needs their attention. Alerts and notifications will be identifiable based on color and its representative icon.

Accessibility

Use the ARIA role="alert" to inform assistive technologies of a time-sensitive and important message that is not interactive. If the message is interactive, use the alertdialog role instead.

Graphic contrast should be a ratio of at least 3:1.

Alerts

warning 3 
<div class="alert alert-danger" role="alert">...</div>
When to use alerts
Component # Description
01 Sealed case
02 Active warrant
03 Past due balance
04 Missing ATN
05 NSF flag
06 Criminal/JV without charges
07 No judge on case except intake
08 Multiple active judges
09 Fees on case not on pay plan
10 Defendant on multiple pay plans - same court/location
Specifications
  • Alerts will feature the "warning" icon to visually distinguish an alert from a notification.
  • The alert will include a badge indicating the number of alerts related to a particular case.
    • <span class="badge badge-dark">...</span>
  • Alerts will remain fixed to retain visibility while scrolling. They will be positioned directly beneath the Case Information Header.
  • Alerts will be clickable with content that expands when clicked. This allows users to expand and contract content as needed.

Notifications

<div class="alert alert-warning alert-dismissible fade show" role="alert">...</div>
When to use notifications
Component # Description
11 Speedy trial approaching
12 Rule 4gi closure
13 Rule 41 closure
14 Past event open
15 Juvenile with MF/MD but no JUCO and not "set for review" status
16 Unserved service documents (except warrants)
17 Closed case with undisposed charges
18 Closed case with no MD code (circuit only)
19 MF/MD docket codes in case but not associated
20 Litigant on multiple open cases in the same court/location
Specifications
  • Notifications will feature the "star" icon to signify that it is important content but not critical.
  • The notification will include a badge indicating the number of notifications related to a particular case.
    • <span class="badge badge-dark">...</span>
  • Notifications will remain fixed to retain visibility while scrolling but with an option to close the notification bar. They will be positioned directly beneath the Case Information Header.
  • Notifications will be clickable with content that expands when clicked. This allows users to expand and contract content as needed.
Guidance

  • Let users know what needs to be done to address alerts.
  • Be polite in messages.
  • Use concise, readable language - avoid jargon and code.
  • Allow a user to dismiss a notification wherever appropriate.
  • Don't include notifications or alerts that aren't related to their current case view.
