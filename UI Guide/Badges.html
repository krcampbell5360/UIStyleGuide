Badges

*Code from Bootstrap

Accessibility

Graphic contrast should be a ratio of at least 3:1.

Static badges

Static badge
<span class="badge badge-primary">...</span>
When to use

A static badge will be used for alerts, notifications, pleas, and dispositions. In the case of alerts and notifications, the primary role of the badge is to provide a number of alerts/notifications on a case. For pleas and dispositions, the badge will display the type of plea/disposition and the corresponding date.

Event parties

The names of parties that are associated with an event will appear in a badge. They will be in two different colors: blue for active parties on the case and gray for associated individuals.

Example - Pill badge for count indicators
4
<span class="badge badge-pill badge-dark">4</span>
Example - Active party
Name
<span class="badge badge-primary">...</span>
Example - Associated parties

Actionable badges

Actionable badge
<a href="#" class="badge badge-secondary">...</a>
When to use

Actionable badges will be use to indicate history and a "Must Appear" status.

Must appear badge

The "Must Appear" badges will be featured on Balance cards where the individual is unable to make a payment online because they are required to appear in court. The gray indicates the inability to pay online.

This badge is an exception to the principle of sentence style labels. The badge uses all caps to signify its importance.

Example
MUST APPEAR
<a href="#" class="badge badge-secondary">MUST APPEAR</a>

The badge is actionable because the user can view information either on hover (for public users) or by clicking which triggers a modal (for internal users).
